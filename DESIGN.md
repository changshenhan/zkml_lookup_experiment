# 分段查找表 + 线性插值实现 Sigmoid

## 设计目标

- 用**查表 + 段内线性**实现 σ(x)，不在电路中把 Sigmoid 展开成大量算术。
- 证明者**不在表中“搜索”x**，而是证明 x 落在某个区间 [x_i, x_{i+1}]，再在电路中算 σ(x) ≈ k_i·x + b_i，仅增加一两个乘法约束。

## 表结构

- **输入范围**：由 `data.json` 得到，在该范围上**非均匀**选取 **1024 个关键点** x_i（按分位点）。
- 对每个区间 [x_i, x_{i+1}]，表项存该段线性拟合的 **k_i** 和 **b_i**（端点线性插值）：
  - k_i = (σ(x_{i+1}) − σ(x_i)) / (x_{i+1} − x_i)
  - b_i = σ(x_i) − k_i·x_i
- 所有 **(x_i, k_i, b_i)** 与输入 x 均在有限域内用整数表示（见 `lookup_table.json` 中的 `scale` 与 `x_int`, `k_int`, `b_int`）。

## 电路侧（理想）

1. 证明者给出 x 所属的段（或由约束推出段索引 i）。
2. 通过 **lookup** 约束从表中取出一行 (x_i, k_i, b_i)。
3. 约束 **x ∈ [x_i, x_{i+1}]**（例如 x ≥ x_i 且 x ≤ x_{i+1}）。
4. 约束 **y = k_i·x + b_i**（一两次乘法/加法）。

当前实现：Sigmoid 由 **ezkl 内置的 Sigmoid 查表** 实现（查表、非算术展开）；`lookup_range` 根据 `lookup_table.json` 的输入范围设置，保证查表满足。**表内容** 为 ezkl 内置的 (input → output) 表，非本方案设计的 (x_i, k_i, b_i) 分段线性表。若需完整按 (x_i, k_i, b_i) 自定义表项，需 ezkl 提供「从文件加载自定义 lookup 表」的接口（当前 Python API 未暴露）。

## 文件

| 文件 | 作用 |
|------|------|
| `build_lookup_table.py` | 从 `data.json` 生成 1024 点与 (k_i, b_i)，写出 `lookup_table.json`（含浮点与整数表项）。 |
| `lookup_table.json` | 表内容与 scale；`run_ezkl_full.py` 可据此设置 `lookup_range`（当前也可用固定 (0,16) 以减小电路）。 |
| `gen.py` | 导出 ONNX，仅含一个 Sigmoid op，由 ezkl 查表实现。 |

## 本方案 vs ezkl 内置 Sigmoid 查表：精度区别

- **ezkl 内置 Sigmoid 查表**  
  - 一般是**均匀或固定步长**的 (x → σ(x)) 表，scale 固定（如 128）。  
  - 精度由**表分辨率**决定：步长大则误差大，例如步长 1/128 量级时，误差约 1/128 或更差。  
  - 不区分数据密集/稀疏区域，整段用同一分辨率。

- **本方案（1024 非均匀点 + 段内线性 k_i·x + b_i）**  
  - **近似误差**：每段用端点线性插值，段内最大误差在区间中点附近；`build_lookup_table.py` 打印的「区间中点最大 |σ(x)−kx−b|」通常可达 **1e-6～1e-10** 量级，远小于 1/128。  
  - **表项精度**：用 scale=2^32 存 (x_i, k_i, b_i)，有限域内量化误差很小。  
  - **非均匀**：在数据密集处点更密，稀疏处更疏，同样段数下在“重要区间”误差更小。

- **当前实际运行**  
  电路里用 **ezkl 内置 Sigmoid 查表**（查表、非算术），执行精度 = ezkl 内置表精度。  
  若 ezkl 将来支持从文件加载自定义 lookup 表，可接入 `lookup_table.json` 的 (x_i, k_i, b_i)，实现「证明 x 在某段 + 计算 k_i·x + b_i」，达到段内线性近似误差 1e-6～1e-10 量级。
